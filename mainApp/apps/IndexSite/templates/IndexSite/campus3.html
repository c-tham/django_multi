Project - The Master of Campus Tracking Services System

2687 lines of code (HTML: 1534 lines, Python: 1153 lines)
Bootstrap 4
Development: June 30 to July 7, 2018

The Master Campus (aka Campus 3) is an upgrade of the Original Campus (aka Campus 2). The application tracks a persons use of services like building and office acccess, meal cards, and parking permits. It also keeps track of addresses, phone numbers, emails, and office extension numbers.

When a person registers to the system, it will generate a college email address. The college email address will be the main login information. The password is stored using bcrypt which it's generating a salt to protect against rainbow table attacks.

The main design of this application is to be able to have multiple applications on one Django. I also add a lot of new features into the program. For example, it will turn red when parking permits are expired, and green when they are still valid.

One of the major bugs for the Original Campus is when database initialize for the first time, the system will be crash when it start running. This is because it is missing values in the database. The only way to resolve the problem is to insert the values into the database manually.

This application have resolved this problem by initializing the values when the values are not found in the database. Moreover, it allows users to delete records (primary and foreign tables). I have included a hack button to allow users to add and remove admin rights.
