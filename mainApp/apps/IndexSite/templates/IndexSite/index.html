Welcome ...

Here's the Django system that supporting multiple applications with their own codes, HTML, CSS, and database. This main application is running it own CSS, the W3.CSS with no database. The second application, the Campus 2, is running Bootstrap 3 CSS with it own database. The third application, the Campus 3, is running Bootstrap 4 CSS with it own database. All the database are running independent with its own tables and structures.

Multiple databases on Django are store web site sessions in flat file or cookie.

Please feel free to try around with my applications. You can start with the Campus 2 first before using Campus 3. Please share with me your thought by giving feedback. If you found a bug, please do not hesitate to let me know. The feedback form is at the bottom of the this page.

Appreciate very much. Thank you!

Project - Campus 3

aws.iotree.com/campus3
github.com/c-tham/django_CampusService3
2687 lines of code (HTML: 1534 lines, Python: 1153 lines)
Bootstrap 4
Development: June 30 to July 7, 2018

The Campus 3 is the upgrade of Campus 2 and more. The main design is to be able to have multiple applications on one Django. I also add a lot of new features into the program. For example, It will turn red when parking permit is expired, and green when it is still valid.

One of the major bug for Campus 2 is when database initial for the first time, the system will be crash when it start running. This is because it is missing values in the database. The only way to resolve the problem is to insert the values into the database manually.

Campus 3 have resolve this problem by initial the values when the values are not found in the database. Moreover, it's allow admin right users to delete records. I have included a hack button to allow users to be add and remove admin right.

Project - Campus 2

aws.iotree.com/campus2
github.com/c-tham/django_CampusService2
1518 lines of code (HTML: 785 lines, Python: 733 lines)
Bootstrap 3
Development: May 26 to 28, 2018

The Campus project is to track a person's use of services like building and office acccess, meal cards, and parking permits. It also keep track of address, personal phone number, personal email address, and office extension number.

When a person registrate to the system, it will generate a college email address. The college email address will be the main login information. The password is store using bcrypt which it's generating a salt to protect against rainbow table attacks.

The system is able to generate unique college ID, unique college email address and unique parking permit ID. I have listed a college ID (under CheatSheet) in the main page for us to login. You can copy and paste it on the login field. Sorry, I will not know your password.

Please use fake information when you registrate to the system. Thanks!
